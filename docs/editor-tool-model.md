# 에디터 도구·인터랙션 모델

정리일: 2026-05-31

좌측 도구(rail)와 하위 옵션, 선택/편집 인터랙션의 현재 합의·구현 상태를 코드 기준으로 정리한 참조 문서다.
설계 논의의 원본 메모는 이슈 #35이며, 이 문서는 거기서 결정된 모델을 구현 기준으로 다듬어 체크인한 스냅샷이다.

## 1. 도구 (rail)

좌측 rail은 4개의 도구를 세로로 둔다. 활성 도구(`EditorMode`)가 지도 클릭의 의미를 결정하며, 기본은 선택이다.

| 도구   | 값         | 역할                    | 하위 옵션                  |
| ------ | ---------- | ----------------------- | -------------------------- |
| 선택   | `select`   | 도형 선택 · 정점 편집   | 없음(기본)                 |
| 그리기 | `draw`     | 새 피처 추가            | 폴리곤 / 패스 / 마커       |
| 경계   | `boundary` | 경계를 선택해 생성·병합 | 행정동 / 법정동 / 우편번호 |
| 반경   | `radius`   | 마커 기준 원형 커팅     | 반경 값                    |

- 도구는 sticky다(동작 후 자동 복귀 없음). 기본 도구는 선택.
- 클릭의 의미는 활성 도구가 결정한다. 무모드 추론은 쓰지 않는다(행정동/법정동 경계가 화면을 가득 채우면 "여백 클릭 = 그리기" 같은 추론이 성립하지 않음).

## 2. 기반 인터랙션 (도구와 무관, 항상 존재)

- **선택**: 도형 클릭 → 강조 + 패널 반영 + 편집 가능 판정. 더하기/빼기/반경 같은 모든 연산의 대상이 된다.
- **정점 편집**: editable 도형을 더블클릭하면 편집 상태로 진입한다(폴리곤/패스는 꼭짓점 드래그·우클릭 삭제, 마커는 점 드래그). ESC/완료 시 선택으로 복귀.
- **배타적 상태기계(계획)**: `interaction: idle | drawing | editing` — 한 번에 하나만. 그리는 중에는 다른 도형 선택을 잠그고, ESC로 진행 중 동작을 취소한다. (구현은 #9 · #11)

## 3. "그리기 = 추가" 개념

그리기는 "새 피처를 추가"하는 도구로 본다. 폴리곤/패스/마커는 "무엇을 추가할지"의 선택이다.

- 폴리곤 = 닫힌 면, 패스 = 열린 선, 마커 = 점 한 개.
- 도형마다 인터랙션이 다르지만(다중 클릭 vs 원클릭) "추가"라는 개념으로 묶이므로 한 도구의 하위 옵션으로 둔다.
- OpenLayers `Draw`는 생성 시점에 타입(Polygon/LineString/Point)을 정해야 하므로, 도형은 그리기 전에 미리 고른다.
- 생성된 피처의 lifecycle은 `Created`.

## 4. 경계

- 하위 옵션 = 행정동 / 법정동 / 우편번호(경계 데이터셋). 줌 레벨에 따른 LOD(시·군·구 ↔ 행정동)는 자동 전환(계획).
- 경계 클릭 시 동작(계획, #13):
  - **생성**: 새 레이어 + 폴리곤 생성
  - **더하기(union)**: 선택된 폴리곤이 있어야 활성
  - **빼기(difference)**: 선택된 폴리곤이 있어야 활성
- 데이터셋 선택(팝업)과 더하기/빼기 동작(선택 폴리곤 대상)은 분리한다.

## 5. 마커 & 반경

- **마커 = 1급 Point 피처**: 씬에 저장되고 host로 전달된다. 그리기-마커로 추가한다. (#41)
- 현재 OpenLayers adapter는 Polygon/MultiPolygon만 변환하므로 Point/Path 렌더링은 #41에서 추가한다.
- **반경**: 선택 폴리곤 + 선택 마커(기준점) 기준으로 원형 커팅한다. 폴리곤과 마커가 겹치지 않으면 무효. 원 안쪽 유지(clip)와 바깥 제거(difference) 중 어느 의미인지는 미정. (#40)

## 6. UI 패턴

- **하위 옵션 팝업 `ToolOptionPopup`**: 도구 버튼 옆(우측)에 붙는 작은 플라이아웃. 제목 + 닫기(X) + 옵션 목록으로 구성한다. 선택해도 닫지 않고 닫기 버튼이나 바깥 클릭으로 닫는다. 경계·그리기가 공용으로 사용하며, 레이어 패널 위(z-50)에 뜬다.
- **선택 인디케이터 `MovingHighlight`**: 선택된 항목 위치로 부드럽게 이동하는 범용 인디케이터. 항목의 실제 위치/크기를 측정하므로 가로·세로 배치 어디서나 동작한다. rail과 옵션 팝업에 함께 쓴다.
- **라벨 규칙**: 하위 옵션이 있는 도구 버튼은 "선택된 옵션 + 도구 의미"를 함께 표시한다. 예) `폴리곤` + `그리기`, `행정동` + `경계`. 처음부터 어떤 도구인지 알 수 있게 모호함을 줄인다.

## 7. 상태 매핑 (store)

- `activeMode`: `select`(기본) / `draw` / `boundary` / `radius`
- `activeDrawShape`: 폴리곤(기본) / 패스 / 마커 — `GeometryKind`의 부분집합 `DrawShape`
- `activeBoundaryKind`: 행정동(기본) / 법정동 / 우편번호
- `selectedFeatureIds`, `hoveredFeatureId`, `activeLayerId`, `dirty`
- (계획) `interaction`, `editingFeatureId`
- 원칙: OpenLayers 객체는 store에 넣지 않는다(adapter의 ref/cache로 관리).
